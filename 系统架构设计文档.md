# 企业智能协同平台 - 系统架构设计文档

---

## 一、技术栈选型

### 1.1 前端技术栈
- **框架**: Vue 3.x (Composition API)
- **UI组件库**: Element Plus
- **构建工具**: Vite
- **状态管理**: Pinia
- **路由**: Vue Router 4
- **HTTP客户端**: Axios
- **富文本编辑器**: Marked + CodeMirror
- **图表可视化**: ECharts 5
- **实时通信**: Socket.IO-client
- **拖拽排序**: Sortable.js
- **视频通话**: WebRTC API

### 1.2 后端技术栈
- **Web框架**: FastAPI
- **异步IO**: Python Asyncio
- **ORM**: SQLAlchemy 2.0 + Asyncio
- **数据验证**: Pydantic
- **身份认证**: PyJWT + Passlib
- **实时通信**: Socket.IO
- **任务调度**: APScheduler
- **数据库连接**: aiomysql + asyncpg
- **容器沙箱**: Docker SDK for Python
- **HTTP客户端**: httpx
- **日志**: structlog

### 1.3 数据存储
- **主数据库**: MySQL 8.0
- **缓存**: Redis 6.x
- **消息队列**: RabbitMQ
- **全文检索**: Elasticsearch 8.x
- **文件存储**: 本地文件系统 / MinIO

### 1.4 基础设施
- **容器化**: Docker + Docker Compose
- **反向代理**: Nginx
- **任务执行**: Docker Container 沙箱
- **爬虫框架**: Playwright

## 二、整体架构设计

### 2.1 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                     前端应用层 (Vue 3 + Element Plus)         │
├─────────────────────────────────────────────────────────────┤
│                     API网关层 (FastAPI)                       │
├─────────────────────────────────────────────────────────────┤
│                     业务逻辑层 (Service Layer)                │
├─────────────────────────────────────────────────────────────┤
│                     数据访问层 (Repository Layer)            │
├─────────────────────────────────────────────────────────────┤
│                     基础设施层 (Infrastructure)              │
├─────────────────────────────────────────────────────────────┤
│                     数据存储层 (MySQL + Redis + ES)          │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 微服务划分

| 服务名称 | 职责描述 | 技术栈 |
|----------|----------|--------|
| auth-service | 用户认证、权限管理、JWT令牌处理 | FastAPI + MySQL |
| im-service | 即时通讯、消息处理、视频通话 | FastAPI + Socket.IO + MySQL |
| ai-service | AI模型管理、NL2SQL、Function Call | FastAPI + Redis + MySQL |
| data-service | 数据采集、ETL清洗、爬虫任务 | FastAPI + Playwright + MySQL |
| admin-service | 系统管理、审计日志、敏感词过滤 | FastAPI + Elasticsearch + MySQL |
| worker-service | 异步任务处理、代码沙箱执行 | FastAPI + Docker SDK + Redis |

## 三、数据库设计

### 3.1 核心表结构

#### 3.1.1 用户表 (users)
```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    phone VARCHAR(20),
    full_name VARCHAR(50) NOT NULL,
    avatar VARCHAR(255),
    department_id BIGINT,
    role ENUM('user', 'data_analyst', 'admin', 'super_admin') NOT NULL DEFAULT 'user',
    status ENUM('active', 'inactive', 'banned') NOT NULL DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    last_login_at DATETIME,
    INDEX (username),
    INDEX (email),
    INDEX (department_id)
);
```

#### 3.1.2 部门表 (departments)
```sql
CREATE TABLE departments (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    parent_id BIGINT,
    sort_order INT DEFAULT 0,
    manager_id BIGINT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES departments(id) ON DELETE CASCADE,
    INDEX (parent_id),
    INDEX (sort_order)
);
```

#### 3.1.3 AI模型表 (ai_models)
```sql
CREATE TABLE ai_models (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
```

```sql
);
```

### 3.2 索引设计原则
- 主键使用BIGINT自增，保证性能
- 常用查询字段建立普通索引
- 唯一字段建立唯一索引
- 全文搜索使用FULLTEXT索引（MySQL）配合Elasticsearch
- 外键建立索引，提升连接查询性能

## 六、安全设计

### 6.1 身份认证
- 使用JWT Token进行身份认证
- Access Token有效期24小时
- Refresh Token支持无感刷新
- Token存储在HttpOnly Cookie中，防止XSS攻击
- 密码使用bcrypt加盐哈希存储，不存储明文

### 6.2 访问控制
- 基于角色的权限控制（RBAC）
- 按部门配置数据访问范围，实现行级数据权限
- API接口统一进行权限校验
- 未授权访问直接返回403错误

### 6.3 数据安全
- AI模型API Key使用AES加密存储
- 数据库连接字符串加密存储
- 所有敏感操作记录审计日志，日志不可篡改
- NL2SQL严格拦截非SELECT语句，只允许查询操作
- SQL查询使用预编译语句，防止SQL注入

### 6.4 代码执行安全
- 自定义Python代码在Docker容器沙箱中执行
- 严格限制容器网络访问，只允许访问企业内部指定数据源
- 限制文件系统访问，只允许读写临时目录
- 设置CPU和内存使用配额
- 执行超时强制终止
- 所有代码执行记录审计日志

### 6.5 敏感内容过滤
- 基于敏感词库对消息内容进行实时检测
- 支持阻断和审计两种处理级别
- 违规消息强制撤回，全端同步
- 对发送人进行禁言/封号处罚

```yaml

version: '3.8'
services:

      - enterprise-net

      - discovery.type=single-node
      - ES_JAVA_OPTS=-Xms1g -Xmx1g

      - es_data:/usr/share/elasticsearch/data

      - enterprise-net

    image: rabbitmq:3-management

      - "5672:5672"
```

## 八、性能优化策略

### 8.1 数据库优化
- 读写分离，主库写从库读
- 常用查询结果缓存到Redis

- 常用查询结果缓存，减少重复NL2SQL调用
- 前端静态资源CDN加速
- 消息列表分页懒加载

### 8.3 异步处理
- 耗时任务（爬虫、ETL、AI生成）放入消息队列异步处理
- 前端通过轮询或WebSocket获取任务进度
- 邮件通知异步发送
- 日志异步写入

### 8.4 并发控制
- 使用Redis实现分布式限流
- 限制单个用户每分钟查询次数
- 防止恶意请求占用系统资源
- WebSocket连接数限制
